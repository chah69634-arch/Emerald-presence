### b · 梦境流动概括条未接入 + 潜意识疑似静态

**现状（已确认）**：分两件事。
- **「梦境流动概括条」**：dream 窗口里没有一条「梦的流动概括」。后端 HUD（`dream_hud.py` derive_hud_v1）输出的是 emotion/boundary/intimacy 等数值条，`dream_summary.py` 的摘要是出梦后才生成、且**契约上明确不注入 prompt/UI**（archive-only）。所以「流动概括条」目前确实没有数据源。
- **「潜意识变动太小像静态」**：面板是 `SubHiddenStatePanel.tsx`，读 `GET /hidden-state-debug`（后端 `core/memory/user_hidden_state*`）。隐性状态有 12h decay + 7d consolidate 的衰减调度（`triggers/hidden_state_decay.py`），单轮 integrate 的增量本就很小，**看起来静态是预期行为**，但需要一次审计确认 reality_behavior 写入是否真的在跑。

**方案**：
1. **先审计潜意识**（半天）：写个一次性脚本 dump 某 uid 的 hidden_state 时间序列，确认每轮 `integrate_event/impression` 有写入、source 分布正常、decay tick 在跑。若增量确实过小，调 integrator 步长或在面板上显示「近 N 次变动 delta」让变化可见。
2. **梦境流动概括条**：需要一个轻量的「梦内实时一句话概括」数据源。两条路线（需你拍板，见文末问题）：

   - 或后端在每轮 dream_turn 后生成一句 ≤15 字的流动概括存进 dream_state 供 `/dream/state` 吐出（需额外 LLM 调用，M）。

**涉及文件**：审计脚本（新增）、`SubHiddenStatePanel.tsx`、`core/memory/user_hidden_state_integrator.py`；概括条按路线定 `dream_hud.py`/`DreamStatusSidebar.tsx` 或 `dream_pipeline.py`+`dream.py`。
**工作量**：审计 S；概括条 S–M。 **依赖/风险**：概括条若走后端 LLM 会增加每轮梦境延迟，建议先做纯前端版。

### f · 群聊里角色只在意「我」、忽略彼此

**现状（已确认）**：仲裁器 `core/stage/arbiter.py` 的发言意愿打分**只看 `transcript[-1]`（最新一条）**，而最新一条通常是你发的；`addressed`（被点名）权重 0.9 最高，`addressed_exclusive` 开启时只有被 @ 的人能答。也就是说角色的「想说」几乎只由「你最后那句」驱动，角色之间不会因为「另一个角色刚说了什么」而接话。叠加人设本身把用户当主要对话对象，就表现为「只在意我」。**部分确实是机制、部分是人设**（你自己也猜到了）。

**方案**（按性价比，建议先 1+2）：
1. **让角色能接彼此的话**：仲裁评分不要只吃 `transcript[-1]`，而是看最近 N 条里「上一个发言者是不是另一个角色」，给「回应同伴」一个 base 分；并允许连续多角色发言（现在 `recency_penalty` 对刚说过的人罚分，方向对，但缺「别人说了我想接」的正向项）。
2. **prompt 侧**：群聊 projection（`core/stage/projection.py`/`context.py`）确认每个角色的上下文里**确实带了其他角色的发言且标注了 speaker**，并在指令里明确「你可以回应在场其他角色，不必每句都对用户」。
3. 调 `talkativeness` / 关掉过强的 `addressed_exclusive` 默认。

**涉及文件**：`core/stage/arbiter.py`、`core/stage/projection.py`/`context.py`、群设置默认值（`models.py`）。先读 `docs/stage.md`。
**工作量**：M。 **依赖/风险**：调高互相接话会增加每轮 LLM 调用与「群聊跑题」风险，需配合 N/M 上限。

3. **meta 监视**：定义要监视什么（前台应用 / 活动窗口？已有 `sensor`/`watch` 路由可能相关）——这块需求最模糊，**先写一页 spec 再开工**。
4. 硬件/蓝牙：维持「等」，不排期。

**涉及文件**：`core/tool_dispatcher.py`、新工具于 `core/tools/`、`config.yaml tools:`、`docs/security_model.md`；前端模式开关。
**工作量**：L（且需先出设计）。 **依赖/风险**：**安全敏感**，危险模式开放系统能力前必须把权限/确认/白名单/审计补齐，宁慢勿错。
