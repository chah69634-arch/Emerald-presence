# CC-02 · 日记按角色分类（c）

> 主战场前端 `D:\ai\Emerald-client`（含 Rust 命令层）；后端 `qq-st-bot` 已就绪，基本不动。
> 开工前读前端 `AGENTS.md` + `docs/frontend-structure.md`。

## 现状（已核对）
- **后端已支持多角色**：`admin/routers/diary.py` 的 `GET /diary/list` 和 `GET /diary/{date}` **都已接受 `char_id` 查询参数**（缺省取 active char），日记按 `get_paths().yexuan_inner_diary(char_id=cid)` 分角色存储。后端无需改。
- **数据链路**：前端 `SubDiary.tsx` → `backend.ts:loadDiaryList()/loadDiaryEntry(date)` → **Tauri 命令** `invoke('load_diary_list')` / `invoke('load_diary_entry',{date})` → Rust `src-tauri/src/lib.rs:355 load_diary_list` / `:376 load_diary_entry` → reqwest 打后端 `/diary/list`、`/diary/{date}`。
- **问题点**：从 `SubDiary` 到 Rust 命令**全程没传 char_id**，所以永远读 active 角色（你看到的叶瑄）。
- **注意**：后端 `emotion` 字段当前**恒为 `None`**，所以 `SubDiary` 现有的「情绪分类 tab」实际上永远是空的（`emotionTabs` 算出来是空集，只剩「全部」）。本次用**角色分类**取代/补上这条分类栏更合理。
- **角色列表来源**：前端已有 `getPromptAssets()`（`backend.ts:76`）返回 characters（含 id），可作为「有哪些角色」的来源；或后端 `GET /characters`（`character.py:91`，返回 `{characters, active_id}`）。优先用 `getPromptAssets`，已有现成解析。

## 实现

### 【前端-Rust】1. 命令透传 char_id `src-tauri/src/lib.rs`
- `load_diary_list`（:355）：加可选参数 `char_id: Option<String>`，拼到 query：当 Some 时给 `/diary/list?char_id=<v>`。照该文件已有 `backend_url` + reqwest 写法。
- `load_diary_entry`（:376）：加 `char_id: Option<String>`，拼到 `/diary/{date}?char_id=<v>`。
- 注意 Tauri 命令参数名 camelCase 约定：前端 invoke 传 `charId`，Rust 形参写 `char_id` 时需 `#[tauri::command]` 的默认 rename（确认项目其他命令惯例，如 `load_diary_entry` 的 `date`、`get_character_avatar` 的 `charId`→`char_id`，照搬同款）。

### 【前端-TS】2. API 层 `src/shared/api/backend.ts`
```ts
export async function loadDiaryList(charId?: string): Promise<DiaryListResponse> {
  return invoke<DiaryListResponse>('load_diary_list', charId ? { charId } : {});
}
export async function loadDiaryEntry(date: string, charId?: string): Promise<DiaryEntry> {
  return invoke<DiaryEntry>('load_diary_entry', charId ? { date, charId } : { date });
}
```

### 【前端】3. SubDiary 加角色分类栏 `src/windows/chat/components/SubDiary.tsx`
- 组件加载时用 `getPromptAssets()` 拉角色列表（id + 显示名），构造一条**角色 tab 栏**（放在现有 filter bar 那一行，或其上方独立一行）。默认选 active 角色。
- 维护 `activeCharId` state；切换角色时用 `loadDiaryList(activeCharId)` 重新拉列表。
- `openEntry` 打开详情时，`DiaryDetailPane` 也要带上 char_id：`loadDiaryEntry(item.date, activeCharId)`。把 `DiaryDetailPane` 的 props 从 `{date}` 改 `{date, charId}`。
- 现有的「情绪分类 tab」因 emotion 恒 null 实际为空，可：
  - 方案 1（推荐）：直接用角色 tab 替换那条情绪栏；
  - 方案 2：保留情绪栏但隐藏（等后端将来填 emotion 再启用），新增角色栏在其上。
  二选一，在 PR 描述里说明。
- 角色显示名：用 prompt assets 里的 name；没有则回退 char_id。可选给每个角色 tab 加头像（`getCharacterAvatar(charId)`，已有）。

### 可选增强（后端，做不做都行）
- 若想要「一次读全部角色再本地分组」，给 `/diary/list` 加 `char_id=all` 分支：遍历 `data/runtime/.../{char_id}/.../diary/` 各角色目录聚合，每条带上 `char_id` 字段返回。前端则一次拉全量、本地按 char_id 分组。**不做也行**，逐角色按 tab 拉更简单。

## 验收
- 日记面板顶部出现角色分类栏，列出所有角色；切换角色能看到对应角色的日记列表与正文。
- 切到没有日记的角色 → 显示「他还没开始写日记。」空态，不报错。
- 详情浮窗读的是所选角色的当天日记（不串角色）。
- `npm run build` 通过，无 console 报错；Rust `cargo build`（或 `npm run tauri build` 的编译步）通过。
