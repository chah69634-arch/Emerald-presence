## 1. 桌宠行为环（→ 施工 11a，纯前端）
目标：没消息时也让人感到"它在"。**state 驱动，不是 message 驱动**（reference doc 的核心原则）。
- 在 `PetWindow` 加一个行为环：按 `snapshot.mood / presence / thinking` 驱动粒子幅度、偶发微动作、idle 文案轮换（非台词，氛围词）。
- presence=away → 收敛/变暗；idle → 安静微动；active → 正常。
- 复用老桌宠 `particle.py` 的 `set_emotion/set_volume` 思路（新端 ParticleCanvas 已是 React 版，扩参即可）。
- **不硬编码台词**：任何"说话"仍走后端（见 §3 触发源），前端只管视觉/氛围。

## 2. 鼠标交互：害羞躲避 + 蹭（→ 施工 11b，纯前端）
移植 `mouse_interact.py` + `mouse_tracker.py` 的设计到 Tauri：
- 取全局鼠标坐标（Tauri 有 window/cursor API；或前端定时轮询）。
- **害羞躲避**：shy 状态下鼠标靠近桌宠窗 → 用 `getCurrentWindow().setPosition()` 把窗推离光标（老桌宠是窗口几何位移，新端同理）。
- **蹭鼠标**：随机间隔（`bother_minutes_min/max`）触发一次靠近/蹭的动作。
- **钉住**：按住 `hold_still` 键（老桌宠默认 Ctrl）时不躲。
- 状态（shy / pinned）可由 mood 或后端 snapshot 驱动。
- 注意：`PetWindow` 现在左键 `startDragging`——躲避/拖拽要协调（如 pinned 时才允许拖，或区分触发区）。

## 3. 存在感弹窗（"假报错"式）— 核心，你点名要的（→ 施工 11c）
把老桌宠 `yandere.py`（病娇弹窗：高活跃 + 长时间无互动触发、连环弹窗、禁交互、ESC/工具退出、台词走 `send_to_channel`）**重写为 Tauri 窗**，皮肤换成**"假系统报错弹窗"**风格——用户生气/被无视时叶瑄借"报错框"找存在感。

### UX 设计
- 一个或多个 Tauri 子窗（label 如 `nag`），无边框/置顶，画成**仿系统错误对话框**（标题栏 ⚠、"错误"图标、确定/取消按钮，但内容是叶瑄的话）。
- 内容**走后端 LLM**（情景提示词，不硬编码）：后端给一个"你想刷存在感/有点闹脾气"的框定，叶瑄自由发挥一句。
- 退出：明确的关闭键（ESC / 点确定）一定能关；**保留硬退**（参照梦境不变量精神，别做成关不掉的真恶意弹窗）。可保留老桌宠"关一弹十"的俏皮，但**必须有一个总的强制全关**（如长按 ESC 或托盘菜单）。

### 触发设计（关键：别变成骚扰）
- 触发源放**后端 scheduler proposer**（新增 `presence_nag` 触发器，照 `dream_exit`/`overflow` 结构），条件参考老桌宠：`activity_level=high` 且**用户无互动超过 N 分钟**（`yandere_minutes`，老桌宠默认 60）且**情绪偏负/被冷落**。
- 经 gating 状态机/冷却/DND/活跃窗口过滤（复用现有机制，避免半夜弹、避免高频）。
- 命中后经 WS 推一个**新 action 类型**（如 `presence_nag`，带 LLM 台词）→ 前端弹"假报错"窗。
- **强开关**：设置里给一个"允许存在感弹窗"总开关，默认……（见 Q）。

### ⚠ 复用 09 的教训：action 名两端必须一致
09 审计发现后端 `_push_desktop_action` 的 type 与客户端 ws.ts case 名**有错配**（minimize/notify/play_pause）。新增 `presence_nag` action 时，**后端 push 的 `type` 与 `ws.ts` 的 case 名、Tauri 命令名三处必须对齐**，并加一个端到端冒烟测试。

## 4. lifecycle 接线（→ 施工 11d，可选/低优先）
- 离开检测 / 开机问候 / 每日触发：老桌宠在前端 `lifecycle.py` 做；新架构**主动开口已归后端 scheduler**。建议：检测（away/wake）在前端或 sidecar 出信号，"说什么"由后端 proposer 决定，前端只渲染。避免前后端各做一套。
