## Part A · 潜意识审计（b-1）

### 已核对的根因（重要，先看）
用户反馈"潜意识变动太小像静态数据"——审计后**基本属实，根因明确**：

- 现实回合**没有任何写入 hidden_state 的路径**。`core/memory/user_hidden_state_integrator.py` 里的 `integrate_event / integrate_impression / integrate_event_and_save / integrate_impression_and_save` **全仓零调用**（已 grep 确认，`core/pipeline.py`、`post_process`、`fixation_pipeline` 均不调）。
- 运行时唯一会动 hidden_state 的只有三处：
  1. **出梦 afterglow**：`core/dream/dream_exit_afterglow.py` → `integrate_afterglow_and_save`（只在做梦并出梦时）；
  2. **12h 衰减** `triggers/hidden_state_decay.py` → `apply_time_decay`；
  3. **7d 基线收敛** → consolidate。
- 结论：**不做梦的话，hidden_state 只会随时间朝基线衰减，没有任何现实对话把"成长"喂进去** → 看起来静态。这不是 bug，是 **Phase 3 写入链（现实侧 integrator）实现了但从未接进回合管线**（属于被搁置的接线，不是坏掉）。

### 这部分的交付（两件，均低风险）

#### 1. 写一份审计脚本 + 结论 `scripts/audit_hidden_state.py`
- dump 指定 uid/char 的 `hidden_state.json` 全字段 + 每个字段的 `last_update_source` + `body_memory` 列表 + 时间戳。
- 跑一段时间序列对比（或对比"聊天前/聊天后"两个快照）证实：现实对话后字段无变化、`last_update_source` 永远只是 `time_decay` / `dream_afterglow` / `init`，从不出现 `reality_behavior`。
- 把结论写进 `docs/known-issues.md`（新增一条），标注状态 `boundary-doc-needed`：现实侧 hidden_state 写入链未接，需先定"哪些现实信号映射到哪种 RealityEventType"再接。

#### 2. 面板可视化让"变动可见" `Emerald-client/src/windows/dream/components/SubHiddenStatePanel.tsx`
> 目的：即便不接写入链，也让你一眼看出"它现在只被衰减驱动"，并在将来接了写入链后能直接看到变化。
- 面板每个字段已显示 `last_update_source`（后端 `/hidden-state-debug` 已返回 `last_update_source`、baseline、current/deficit、body_memory）。强化展示：
  - 高亮 `last_update_source`（用现有 `SOURCE_HUE` 配色），让"全是 time_decay"一目了然。
  - 显示 `current − baseline` 的 delta 数值/小箭头，强调当前偏离基线多少。
  - 可选：前端轮询两次快照算"自上次查看以来的变化量"，无变化显式标"无变化"。
- 纯展示，不改后端写入逻辑。

#### 3.（可选 follow-up，不在本次施工）接现实写入链
- **不要在本任务里顺手接**——它要改记忆契约 + source 守卫（`_assert_not_long_term`、`UpdateSource.REALITY_BEHAVIOR`），且要定"对话里什么信号→`integrate_event(RealityEventType.x)`/`integrate_impression`"。
- 本任务只**产出一份接线方案 doc**（`cc-tasks/08b-hidden-state-接现实写入.md`，列：在 `post_process`/`fixation_pipeline` 哪个点调 `integrate_*_and_save`、事件类型映射、并发与 source 守卫注意），交给你决定是否排期。

## Part B · 梦境流动概括条（b-2，纯前端，已拍板走前端版）

### 现状（已核对）
- `GET /dream/state` 已返回 HUD 字段：`emotion_label`、`scene_label`、`emotion_tension`、`boundary_intrusion`、`intimacy_tendency`、`obsession`、`dream_stability`、`dream_depth`、`physiological_arousal`（均 0–100，EMA 平滑，`dream_hud.py` 派生）。
- `DreamStatusSidebar.tsx` 已渲染 HudMetric/HudLabel，有 `isActive`（DREAM_ACTIVE/EXIT_REQUESTED）判定——概括条挂这里最自然。
- 「流动概括条」当前缺的只是把这些字段**拼成一句会随梦流动的概括**，无需任何后端改动/额外 LLM。

### 实现 `Emerald-client/src/windows/dream/components/DreamStatusSidebar.tsx`
1. 在侧栏顶部加一条「流动概括」：用 `scene_label` + `emotion_label` + **趋势箭头**组合成一句，例如：
   - 模板：`「{scene_label} · {emotion_label}」` + 趋势（如 `情绪↑ 边界→ 深度↑`）。
   - 趋势来源：组件保存上一次轮询的 HUD 值，本次对比 `emotion_tension`/`boundary_intrusion`/`dream_depth` 的增减 → ↑/↓/→（设一个小阈值如 ±3 避免抖动）。
2. 用现有 `useDreamState` 的轮询数据；在 `useRef` 里存 prev 值算 delta。非 active（不在梦里）时隐藏或显示「—」。
3. 样式跟随梦境主题 token（复用 `dream-hud__*` class / DreamTokens.css），与现有 HudLabel 风格统一。
4. 可选润色：根据 `dream_stability` 高低给概括条不同色调（稳定=冷静色，低稳定=波动色），强化"流动感"。

### 验收
- 梦境进行中，概括条随每次 state 轮询更新文字与箭头；情绪/边界/深度变化方向正确。
- 不在梦里时不报错、显示空态。
- 零后端改动、零额外延迟。
