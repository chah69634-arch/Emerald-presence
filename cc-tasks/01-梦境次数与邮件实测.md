- 需求：**只统计"有效"梦——该场梦里用户对话轮数 > 3 才计入**，≤3 视为测试/手误丢弃。

### 实现

#### 【后端】1. 计数函数 `core/dream/dream_log.py`
新增纯函数（只读 archive，零副作用，所有路径走 `get_paths()`）：

```python
# 有效梦阈值：用户发言条数 > 此值才计入（≤ 视为测试/手误）
VALID_DREAM_MIN_USER_TURNS = 3

def count_valid_dreams(*, char_id: str = "yexuan") -> dict:
    """统计该角色的有效梦境数。

    遍历 archive 目录，每个 dream_*.jsonl 数其中 role == 'user' 的行数，
    > VALID_DREAM_MIN_USER_TURNS 才计入。
    返回 {"total_valid": int, "total_archived": int, "last_dream_at": float|None}。
    纯只读；文件损坏/缺失时跳过该文件并 logger.warning，不抛。
    """
```
要点：
- 用 `glob("dream_*.jsonl")`；逐行 `json.loads`，`try/except` 包住单行解析，坏行跳过。
- `last_dream_at` 取所有计入文件里最大的时间戳（append_turn 写入的 `ts`/`time` 字段——**先确认 append_turn 实际写的时间字段名**，对齐它，没有就用文件 mtime）。
- 阈值用模块常量，便于以后调。

#### 【后端】2. 端点 `admin/routers/dream.py`
照该文件已有 `_owner_uid()` + 从 pipeline 取 `active_character_id` 的写法，新增：

```python
@router.get("/dream/stats", summary="梦境次数统计（只读，有效梦 > N 轮）")
async def dream_stats_get(_auth=Depends(verify_token)):
    from core.pipeline_registry import get as _get_pipeline
    from core.dream.dream_log import count_valid_dreams
    pl = _get_pipeline()
    char_id = (pl._active_character_id if pl else None) or "yexuan"
    return count_valid_dreams(char_id=char_id)
```
- 只读、不碰 dream_state、不触发任何 pipeline。
- 注意 archive 是按 `char_id` 组织（文件名不含 uid），单用户系统按 active char 统计即可。

#### 【前端】3. 接线 `Emerald-client`
- `src/shared/api/backend.ts`：加 `loadDreamStats()` 调 `GET /dream/stats`（照该文件已有 fetch+鉴权 header 的写法）；`src/shared/api/types.ts` 加返回类型 `{ total_valid:number; total_archived:number; last_dream_at:number|null }`。
- `src/windows/dream/components/DreamPrefsPane.tsx`：组件加载时拉 stats，把第 663 行 `value="—"` 改为 `value={stats ? String(stats.total_valid) : '—'}`，`detail` 改 `"VALID DREAMS"`。加载失败保持 `—`（不要报错弹窗）。

### 验收

### 实现

#### 1. 加一个一次性测试入口（不要靠 7 天冷却的真触发器去测）
新建脚本 `scripts/test_mail.py`：
```python
"""手动测试邮件发送。用法：python scripts/test_mail.py"""
import asyncio
from core.mail.mail_sender import send_letter

async def main():
    ok = await send_letter(
        subject="测试来信",
        body_text="这是一封测试信。\n\n如果你收到了，说明邮件链路正常。\n\n——叶瑄",
    )
    print("发送结果:", "成功" if ok else "失败（看上方日志）")

if __name__ == "__main__":
    asyncio.run(main())
```
- 注意 `send_letter` 在 `enabled:false` 时直接返回 False。所以测试步骤是：
  1. 临时把 `config.yaml` 的 `mail.enabled` 改 `true`；
  2. 确认依赖：`pip install aiosmtplib`（requirements 里若没有则补上）；
  3. 跑 `python scripts/test_mail.py`，去 chah69634@gmail.com 收件箱确认收到、标题带【来信】、发件名显示叶瑄；
  4. 失败看日志（常见：Gmail 应用专用密码失效、587 被网络拦、需 `start_tls`——代码已用 start_tls）。

#### 2. 开启策略（测通后）
- 真信由 `letter_writer` 触发器在满足条件时 7 天最多发一封。确认你要的是这个节奏。
- 若要让它真正会发，保持 `mail.enabled: true` 即可；不想被动发就测完改回 `false`，只在需要时开。

#### 3.（可选硬化，推荐）密码移出明文
- 在 `core/config_loader.py` 读 `mail.smtp_password` 时，优先取环境变量 `YEXUAN_MAIL_PASSWORD`，回退 config 值。
- `config.yaml` 里 `smtp_password` 留空或占位，密码放环境变量。
- 这样即使将来 config 误提交也不带密码。**仅在你愿意时做，不阻塞测试。**

### 验收
- `python scripts/test_mail.py` 在 `enabled:true` 下真实收到邮件。
- `enabled:false` 时跑脚本打印"失败"且日志为 `[mail] disabled, skipping send`（证明开关生效）。
