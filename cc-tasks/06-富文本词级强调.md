# CC-06 · 富文本：语境/角色驱动的词级强调（j）

> 跨双仓，但**必须同一份一起落**（后端定标签契约 + 前端渲染 + 模型指令三件套，拆开会半截不通）。
> 先读后端 `core/narrative_parser.py` 全文 + 前端 `src/windows/chat/components/ChatPanel.tsx` 的渲染段 + `chatDisplay.ts`。

## 目标（按用户拍板）
让回复里**部分词**按语境/角色性格变样式：关键词**变红**、**变大/变小**。活泼角色话语更灵动（多用强调），沉稳角色偶尔说重话时把重点词变红。这是**段内词级**强调，比段类型更细。

## 现状（已核对，关键）
- 已有完整 **NMP（Narrative Message Protocol）**：
  - 后端 `core/narrative_parser.py`：`parse_narrative_segments(reply)` 把 `<say>/<do>/<env>/<feel>`（或 markdown `*action*`/`_feel_`/`> env`）解析成 `segments=[{type, text}]`。`KNOWN_TAGS={say,do,env,feel}`；`_ALL_TAG_RE=</?[a-zA-Z]\w*>` 把**所有** xml 式标签从 `content`（纯文本）里剥掉；永不抛（兜底单段 narration）。
  - 桌面端走 WS `message_segments` 事件（前端 `types.ts:13`，`NarrativeSegment{type,text}`，type ∈ say/do/env/feel/narration）。
- 纯文本路径已"自动洁净"：`turn_sink._fanout`（`core/turn_sink.py:116`）发任何通道前先 `strip_render_tags(assistant_text)`（同款 `</?[a-zA-Z]\w*>` 通用剥除）；记忆路径 `memory_text=_scrub(_strip_tags(...))`（:201 附近）也剥。
  - **含义**：**任何新加的 `<xxx>` 标签会被这些通用正则自动从 QQ/mobile/记忆/hidden_state 路径剥掉**——这正是我们想要的（重点词样式只该出现在桌面富渲染，不该污染记忆和别的通道）。
- 前端：流式期 `renderStreamingContent`（`ChatPanel.tsx:221`）把 `<tag>` 显示成小徽章；定稿走 `normalizeChatDisplayText(msg.segmentedContent ?? msg.text)`（仅做 `*x*`→（x））。**当前没有词级样式渲染。**

## 设计：新增 3 个**段内 inline 样式标签**，桌面 segments 路径保留并渲染，其他路径自动剥
选定标签（小写、单词，命中通用 tag 正则 → 别的通道自动剥除）：
- `<hl>…</hl>` 高亮/强调（默认渲染为主题强调色，"变红"）
- `<big>…</big>` 放大
- `<sm>…</sm>` 缩小

> 为什么用这组：它们被现有 `_ALL_TAG_RE` / `strip_render_tags` 当普通标签剥掉，所以 **QQ / mobile / memory / hidden_state 全部零改动就保持纯文本**。我们只需在**桌面富渲染这一条路径**上"放行 + 渲染"它们。

### 【后端 qq-st-bot】

#### 1. `core/narrative_parser.py`：段内保留白名单 inline 标签
- 定义 `INLINE_STYLE_TAGS = frozenset({"hl","big","sm"})`。
- 解析 segment 时，**段的 `text` 字段保留**这三种 inline 标签原样（开/闭都留），**剥除其他所有** xml 标签（用一个"白名单之外才剥"的正则，而不是无脑 `_ALL_TAG_RE`）。
  - 注意：`content`（marker-stripped 纯文本，供非桌面/统计用）**仍然全剥**（包括 hl/big/sm）——保持现状，不动。
  - 即：只有进入 `segments[i].text` 的内容保留白名单 inline 标签；`content` 一律纯文本。
- `KNOWN_TAGS`（say/do/env/feel）语义不变；`<hl>` 等是**段内**标记，不作为段类型，不参与段切分（确保 `<say>正文<hl>词</hl>更多</say>` 仍是一个 say 段，text 含 inline 标签）。
- 保持"永不抛、兜底"的契约；加单测：`<say>我<hl>很</hl>想你<do>笑</do>` → say 段 text 含 `<hl>`，do 段不含，`content` 为纯「我很想你」（无任何标签）。

#### 2. 确认桌面 segments 不被二次剥
- 排查 segments 的产生与下发路径（`core/output/` / `desktop_output` / `desktop_pipeline` / `response_processor`），确保 segment.text 在送到桌面前**没有再被** `strip_render_tags` / `_ALL_TAG_RE` 洗掉 inline 标签。若发现 desktop 最终是用"纯文本 text"而非 segments 渲染，则二选一：
  - (推荐) 桌面定稿渲染切到 **segments**；
  - 或在 `turn_sink` fanout 里给 **desktop 通道单独**用"保留白名单"的 strip（其余通道维持全剥）。`_fanout` 已有 `if name in ("desktop","mobile")` 分支可挂。
- QQ/mobile/memory 路径**不要动**，维持全剥。

### 【前端 Emerald-client】

#### 3. inline 样式渲染器（白名单 + 转义，禁止裸 HTML 注入）
- 新建 `src/windows/chat/inlineStyle.tsx`（或并入 `chatDisplay.ts`）：`renderInlineStyled(text: string): ReactNode[]`
  - 词法切分，只认 `<hl>…</hl>`/`<big>…</big>`/`<sm>…</sm>`；其余 `<...>` **当普通文本转义显示**（绝不当标签）。
  - 映射：`hl`→`<span style={{color:'var(--accent, oklch(0.55 0.18 25))', fontWeight:600}}>`、`big`→`fontSize:'1.18em'`、`sm`→`fontSize:'0.85em', opacity:0.8`。颜色用主题变量，跟随日/夜主题。
  - 限制：不嵌套（或最多一层）、单标签内容长度上限、未闭合标签按纯文本处理。**不要用 `dangerouslySetInnerHTML`**，全部用 React 节点拼。
- 定稿渲染接入：把 `normalizeChatDisplayText(...)` 输出再过 `renderInlineStyled`（顺序：先 `*x*`→（x）等归一，再解析 inline 样式）。对 say/narration 段启用；do/env/feel 段是否启用按观感定（建议 say 启用即可）。
- 流式期：`renderStreamingContent` 现在会把 `<hl>` 也显示成徽章。可选优化：流式时对白名单 inline 标签**不显示徽章、直接当文本/样式**（避免说话中途蹦出 "hl" 小框）。最简做法：在 `renderStreamingContent` 的标签分支里，若 tag ∈ {hl,/hl,big,/big,sm,/sm} 则并入文本缓冲而非 StreamTagBox。

### 【模型侧】4. 教模型用这组标签 + 角色性格映射
- 定位 NMP 输出格式是在哪教模型的（模型已会输出 `<say>/<do>`——查 `characters/reality/jailbreaks/base.json`、角色卡、author notes、或 `prompt_builder` 的输出格式层）。在同处补一段（措辞可打磨）：
  ```
  你可以在 <say> 内对个别词做轻微强调，让语气更有表情：
  · <hl>词</hl> 强调/重音（会显示为醒目色）
  · <big>词</big> 放大　· <sm>词</sm> 缩小
  克制使用：一句话最多强调 1–2 处，不要整句套标签。
  ```
- **角色性格映射**：放进角色卡/author note 而非全局，让每个角色自带倾向：
  - 活泼角色：「你说话生动，可适当多用 <hl>/<big> 让语气灵动跳脱。」
  - 沉稳角色：「你话不多，仅在说重话/点关键时，偶尔用 <hl> 标一两个重点词。」
- 注意 `core/memory/user_hidden_state.py:58` 已规定"render tag 不可作为隐性状态证据"——本改动天然满足（hidden_state 走的是全剥后的纯文本）。

## 验收
- 模型输出带 `<hl>/<big>/<sm>` 时，桌面气泡里对应词变色/变大变小，其余文字正常；非白名单 `<xxx>` 原样转义显示、不破坏渲染。
- 同一条回复在 QQ/mobile/记忆/日记里是**纯文本**（无任何标签残留）——验证 `strip_render_tags`/`content` 全剥仍生效。
- hidden_state / episodic 等记忆内容不含样式标签。
- 流式过程中不因 inline 标签蹦出突兀的 "hl" 徽章（若做了流式优化）。
- 活泼 vs 沉稳两个角色的强调密度有可感差异。
- 后端 `pytest`（narrative_parser 新单测）+ 前端 `npm run build` 通过。
