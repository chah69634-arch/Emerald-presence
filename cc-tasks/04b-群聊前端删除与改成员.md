## 实现

### 1. API 绑定 `src/shared/api/backend.ts`
照现有 group 函数写法新增：
```ts
export async function deleteGroup(id: string): Promise<void> {
  await invoke('group_delete', { id });
}
export async function patchGroupRoster(id: string, roster: string[]): Promise<GroupDetail> {
  return invoke<GroupDetail>('group_patch_roster', { id, roster });
}
export async function patchGroupSettings(id: string, settings: {min_responders?:number; max_responders?:number}): Promise<GroupDetail> {
  return invoke<GroupDetail>('group_patch_settings', { id, settings });
}
```

### 2. Rust 命令 `src-tauri/src/lib.rs`
照 `group_get`/`group_send`（reqwest + `authorized_request` + `backend_url`）新增：
- `group_delete(id)` → `DELETE /group/{id}`
- `group_patch_roster(id, roster)` → `PATCH /group/{id}/roster`（body `{roster}`）
- `group_patch_settings(id, settings)` → `PATCH /group/{id}/settings`
注册进 `invoke_handler!` 列表 + backend allowlist 那张表（加这三条路由/方法）。

### 3. 删除群入口 `GroupListPanel.tsx`
- 群列表每行右侧加一个删除按钮（复用 `Btn danger` 或小垃圾桶图标）。
- 点击 → 二次确认（"删除『群名』？聊天记录一并清除，不可恢复。"）→ `deleteGroup(id)` → 成功后刷新列表（`load()`）。
- 删除正在查看的群时回退到列表（若当前 `groupView===该id` 则 `onBack`）。

### 4. 群设置面板（改成员 + 改人数）
在 `GroupChatPanel` 顶部加一个「群设置」入口（齿轮图标），打开一个面板/浮层：
- **成员管理**：列出所有角色（`getPromptAssets()` 拿全量），勾选当前 roster；增/减后调 `patchGroupRoster(id, newRoster)`。
  - 至少保留 1 人；roster 改变后若 max>成员数，后端会自动夹紧（前端也同步显示）。
- **回应人数 N/M**：两个滑条（min/max），上限跟随当前 roster 长度（参照建群时 `Math.min(selected.length, 6)`），调 `patchGroupSettings(id, {min_responders, max_responders})`。
- 保存后用返回的 `GroupDetail` 刷新本群视图（roster/settings）。
- 复用 `GroupListPanel` 建群里已有的角色勾选 + 滑条 UI，抽成共享组件更省事。

### 5.（可选）群聊偏好默认栏
CC-04 后端若加了 `config.group_defaults`，可在偏好里加一个「群聊默认 N/M」设置；没加就跳过，不阻塞。

## 验收
- 群列表能删除群（带确认），删除后列表刷新、后端文件+transcript 清除；删当前群会回退。
- 进群后能开「群设置」改成员（加/减角色）和 N/M，保存即生效，max 不超过成员数。
- `deleteGroup/patchGroupRoster/patchGroupSettings` 三个命令端到端通（Rust 命令名 + allowlist 对齐）。
- `npm run build` + Rust 编译通过，无 console 报错。
