# CC-12a · 活动陪伴对话统一：给象棋/阅读补 chat + grounding（待办 e-B/C）

> 跨双仓。把"五子棋已有的边做事边聊"推广到象棋、阅读。先读 `cc-tasks/audit-activity就绪度.md` + `docs/activity-session.md` + `docs/gomoku-activity.md`。

## 模板（已核对，照它做）
五子棋是标杆：
- 端点 `POST /gomoku/chat`（`admin/routers/gomoku.py:227`）→ `gomoku_companion.generate_reply(char_id, uid, session_id, state, user_message)` 返回 `(reply, control, grounding)`。
- **只写 activity transcript**（`core/activity/transcript.py`，user_chat/assistant_chat），不写 short_term / event_log / user_hidden_state，不改棋盘。
- grounding：`core/activity/gomoku_grounding.py:build_gomoku_grounding_facts(state)` 产出**确定性事实**（成几连/挡截/靠中心/相邻子数），作为 `<game_facts>` 注入陪伴 LLM，让叶瑄"真的看懂这手"。
- `core/activity/session.py` + `transcript.py` 是**通用**的，象棋/阅读可直接复用。

## 现状缺口
- 象棋：有 start/state/move/legal_moves/close，**无 `/chat`、无 grounding**。
- 阅读：有 start/state/page/turn_page/close，**无 `/chat`、无 grounding**。
- 前端 `ActivityCompanionPanel.tsx` **只 import `gomokuApi`**，写死五子棋。

## 实现

### 【后端】1. 象棋陪伴 `admin/routers/chess.py` + `core/activity/chess_grounding.py`(新) + `chess_companion.py`(新)
- 加 `POST /chess/chat`，照 gomoku_chat 结构（校验 active session、只写 transcript、不改棋盘）。
- `build_chess_grounding_facts(state)`：从 FEN/move_history 用 `python-chess`（象棋后端已依赖）派生事实——轮到谁、上一步、是否将军/吃子/升变、大致子力优劣、是否有简单战术（可先给"将军/吃子/普通"三档，别做成引擎评估）。作为 `<game_facts>` 注入。
- `chess_companion.generate_reply(...)` 复用 gomoku_companion 的骨架（同一 prompt 模式 + grounding 注入 + 调 LLM）。**建议把 gomoku_companion 里与棋种无关的部分抽成共享 `activity_companion` helper**，象棋/五子棋各传自己的 grounding。

### 【后端】2. 阅读陪伴 `admin/routers/reading.py` + `core/activity/reading_grounding.py`(新)
- 加 `POST /reading/chat`，同上（只写 transcript）。
- `build_reading_grounding_facts(state)`：当前页码/总页数 + 当前页文本的简短摘要（截断或一句话），作为 `<page_context>` 注入，让叶瑄"知道读到哪、这页讲什么"。注意页文本可能长 → 截断 + 不入记忆。

### 【前端】3. CompanionPanel 按活动适配 `Emerald-client/src/windows/activity/components/ActivityCompanionPanel.tsx`
- 现在写死 `gomokuApi`/`GomokuGroundingFacts`。改为按 `activityId`（gomoku/chess/reading）选对应 `xxxApi.chat` + grounding 渲染。
- `src/shared/api/activity-api.ts` 加 `chessApi.chat` / `readingApi.chat`，照 `gomokuApi.chat` 写法。
- grounding 提示渲染做成各活动一个小适配函数（五子棋的 `buildUserMoveHint` 已是范例）；象棋显示"将军了/吃了对方的车"，阅读显示"读到第 X/Y 页"。
- dream_seed 已有独立 `/chat` 面板，本次**可不动**（或后续统一，见可选）。

## 验收
- 下象棋时能和叶瑄聊天，且她的话能体现盘面（将军/吃子/轮到谁）。
- 一起看书时能聊天，叶瑄知道读到第几页、这页大概讲什么。
- 三种活动的陪伴对话只写 activity transcript，不污染 short_term/hidden_state/event_log（跑 contract smoke test 确认 memory policy 不变）。
- CompanionPanel 按活动正确切换 api/grounding，无串台。
- 后端 `pytest`（补 chess/reading chat + grounding 用例）+ 前端 build 通过。
