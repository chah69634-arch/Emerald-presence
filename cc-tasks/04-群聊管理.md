## 现状（已核对）
- 后端 `admin/routers/group.py` 路由仅：`GET /list`、`POST /create`、`GET /{id}`、`POST /{id}/send`、`GET /{id}/history`、`GET /{id}/settings`、`PATCH /{id}/settings`。
  - **无删除路由**；`PATCH settings` **只改 `min_responders`/`max_responders`**，不能改 roster。
- `core/stage/store.py`：有 `create_stage / save_stage / load_stage / close_stage(软关闭) / load_transcript / append_transcript`，**无硬删除**。
- `core/stage/models.py`：`Stage` 是 `@dataclass(frozen=True)`，`roster: tuple[str,...]`（改要用 `dataclasses.replace`）；`StageSettings` 含 `min_responders=1 / max_responders=2 / max_ai_chain_depth=2 / addressed_exclusive=False / transcript_limit=200 / talkativeness / keywords`。
- 前端 `windows/chat/`：`ChatWindow.tsx` 同时持有 `ChatPanel`(单聊主界面) / `GroupListPanel` / `GroupChatPanel`，靠状态切换；`Ribbon.tsx:121` 的「群聊」按钮 `onGroupOpen` 进群视图。`GroupChatPanel` 已有「返回群列表」按钮(:458)。`GroupListPanel` 已定义 danger 按钮样式但**未用**（无删除入口），建群滑条 min 1 / max≤6。

## 【后端 qq-st-bot】Part 1

### 1. 硬删除 `core/stage/store.py` + `admin/routers/group.py`
- store.py 加 `delete_stage(group_id: str) -> bool`：删 stage 持久化文件 + 对应 transcript（路径走 `get_paths()`，照 `load_stage`/`load_transcript` 反推目录）。找不到返回 False，不抛。
- group.py 加：
```python
@router.delete("/{group_id}", summary="删除群（连同 transcript）")
async def delete_group(group_id: str, _auth=Depends(verify_token)):
    from core.stage.store import delete_stage
    ok = delete_stage(group_id)
    if not ok:
        raise HTTPException(status_code=404, detail="群不存在")
    return {"ok": True, "deleted": group_id}
```
- 注意正在跑的轮次：删除前若该群 `group_send` 在跑，简单处理＝照常删（下一轮 load 不到自然结束）；在 PR 里说明取舍。

### 2. 可改 roster + N/M 的 PATCH `admin/routers/group.py`
扩展现有 `PATCH /{group_id}/settings`，或**新增** `PATCH /{group_id}/roster`（推荐后者，职责清晰）：
```python
@router.patch("/{group_id}/roster", summary="改群成员（加/减角色）")
async def patch_group_roster(group_id: str, body: dict, _auth=Depends(verify_token)):
    stage = _require_stage(group_id)
    new_roster = body.get("roster") or []
    # 校验：非空、无重复、char_id 存在（照 create_group 的校验）、min<=max<=len(roster)
    from dataclasses import replace
    updated = replace(stage, roster=tuple(new_roster), updated_at=now_iso())
    # 若 max_responders > 新 roster 长度，自动夹紧 settings
    if not save_stage(updated):
        raise HTTPException(status_code=500, detail="保存失败")
    return {**_summary(updated), "settings": _settings_dict(updated.settings)}
```
- roster 变更后若 `max_responders > len(roster)`，把 settings 夹紧到 `len(roster)` 再存。
- `min/max` 仍走已有 `PATCH /{group_id}/settings`（已能改）。

### 3. （配合前端偏好）群聊默认值来源
- 让 `create_group` 在 `settings_raw` 缺省时，从一个**群聊偏好默认**读取 `min/max/addressed_exclusive`。简单做法：config 加 `group_defaults:` 段；`create_group` 未传 settings 时用它。这样前端「群聊偏好栏」改的是 config 默认，建群即生效。

## 验收
- 建群 → 列表出现；点删除 + 确认 → 列表消失，后端文件与 transcript 清除。
- 进群聊 → 左上角能一键回单聊主界面；群聊内仍可返回群列表。
- 建群后能加/减成员、改 N/M，且 max 不超过当前成员数。
- 群聊偏好栏改默认后，新建群采用新默认。
- 后端 `pytest`（stage 相关）通过；前端 `npm run build` + Rust 编译通过。
