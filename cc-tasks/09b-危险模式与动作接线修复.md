## 已定决策
- 安全/危险：**全局模式开关 + 保留单工具 `enabled` 细粒度**。
- danger 模式**开启后超时自动回 safe**（建议 2h，设常量）。
- 玩具/桌面动作工具留 `dangerous=False`，靠模式闸约束；**仅 `device_shutdown`/`device_sleep` 保留 `dangerous=True`**（双保险，danger 模式下仍走确认）。

## 现状（已核对）
- 桌面动作工具已注册并实现：`desktop_open_url` / `play_song`(网易云) / `desktop_play_pause` / `desktop_minimize` / `desktop_notify`（category `desktop`，`dangerous=False`）；`device_shutdown`/`device_sleep`（`dangerous=True`，category `system`）。
- `execute()` 已有 origin 闸（`_EXECUTE_ALLOWED_ORIGINS`）、`_is_tool_enabled`（`config.tools.<name>.enabled`，默认 True）、`_is_dangerous_tool`、确认流。
- 下发：tool → `_push_desktop_action(action)` → desktop WS（5s ack）→ 降级文件队列。
- **动作名错配（必须修）**：
  | 后端 `_push_desktop_action` 的 `type` | 客户端 `ws.ts` case | Tauri 命令 | 通不通 |
  |---|---|---|---|
  | `open_url` | `open_url` | `action_open_url` | ✅ |
  | `minimize` | `minimize_window` | `action_minimize_window` | ❌ 名不符 |
  | `play_pause` | `media_play_pause` | `action_media_play_pause` | ❌ |
  | `notify` | `show_notify` | `action_show_notify` | ❌ |
  | `play_netease`(play_song 推) | （无 case） | （无命令） | ❌ 客户端没接 |

## 实现

### 【后端】1. 全局模式闸 `core/tool_dispatcher.py`
- 受限类别常量：`_MODE_RESTRICTED_CATEGORIES = frozenset({"desktop", "system"})`。
- 运行态 mode 读写：存 `data/runtime/meta_mode.json`（走 `get_paths()`，新增 path）：`{"mode":"safe"|"danger", "expires_at": float|None}`。读时若 `expires_at` 已过 → 视为 safe。
- `execute()` 入口加 `_mode_gate(tool_name)`：
```python
spec = _TOOL_REGISTRY.get(tool_name, {})
if spec.get("category") in _MODE_RESTRICTED_CATEGORIES and _current_mode() != "danger":
    return ("现在是安全模式，叶瑄不能操作你的电脑。要先在设置里开启危险模式哦。", None)
```
  放在 origin 闸之后、`_is_tool_enabled` 之前。返回友好文案，不抛。
- `device_shutdown/sleep` 仍 `dangerous=True` → danger 模式下依旧走确认流（不变）。

### 【后端】2. 模式端点 `admin/routers/system.py`
```python
@router.get("/system/meta-mode")   # 返回 {mode, expires_at}
@router.patch("/system/meta-mode") # body {mode, ttl_seconds?}；danger 默认 ttl=7200，到点自动回 safe
```
- PATCH 写 `meta_mode.json`；danger 时 `expires_at = now + ttl`。Bearer 鉴权。

### 【后端】3. 修动作名（与客户端对齐）`core/tool_dispatcher.py`
把 `_push_desktop_action` 的 `type` 改成与客户端一致：`minimize`→`minimize_window`、`play_pause`→`media_play_pause`、`notify`→`show_notify`。`play_netease` 见下客户端补 case。
> 或反向：改客户端 case 名。**二选一，统一即可**——推荐改后端 type 对齐客户端（客户端命令名已和 Tauri 一致，动得少）。

### 【前端】4. 补 `play_netease` 动作 + 模式开关 `Emerald-client`
- `ws.ts` 加 `case 'play_netease'`：调一个新 Tauri 命令 `action_play_netease(song_id)`，用 `tauri-plugin-opener` 打开 `https://music.163.com/song?id=<id>`（或网易云 URL scheme `orpheus://`，若装了客户端）。`actions.rs` 加该命令 + `lib.rs` 注册 + allowlist。
- 设置里加「安全 / 危险模式」开关：调后端 `PATCH /system/meta-mode`；显著提示"危险模式允许叶瑄操作你的电脑（开浏览器/放歌/最小化/通知），N 小时后自动收回"。
- 可显示当前模式与剩余时间。
