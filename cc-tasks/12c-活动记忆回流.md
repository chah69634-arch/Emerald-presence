# CC-12c · 活动记忆回流（让叶瑄"记得我们一起做过的事"）（待办 e-D）

> 后端为主 + **设计决策**。先读 `docs/activity-session.md` + `docs/memory.md`（固化链）+ `cc-tasks/audit-activity就绪度.md`。
> ⚠ 这项改的是记忆契约，**先想清楚再做**——当前活动与主记忆是**有意隔离**的。

## 现状（已核对）
- `core/activity/registry.py` 的 `MemoryPolicy` 默认：`writes_short_term=False / writes_hidden_state=False / writes_event_log=False`，`main_memory ∈ {"deferred","none"}`，并留了 `summary_threshold`。
- 即"一起下棋/看书"**不进主记忆**——叶瑄事后不会"记得我们下过棋"。这是当前设计的隔离边界，不是 bug。
- registry 已预留 `summary_threshold` + `main_memory:"deferred"` 这两个钩子，说明"回流"是预期中的后续。

## 设计决策（先定，再施工）
1. **哪些活动回流？** 建议：对局类（gomoku/chess）和阅读，在**结束（close）且达到 `summary_threshold`**（如对话/回合数 ≥ N，避免点开就关也记一笔）时，生成一条**简短摘要**回流主记忆。dream_seed 已有自己的种子产出链，可不重复。
2. **回流成什么？** 一条 episodic/event_log 级别的"我们今天一起做了 X，过程大致 Y"（情绪/关系向，不是棋谱流水账）。**不回流逐手/逐页原文**。
3. **走哪条链？** 复用现有固化链（`core/memory/fixation_pipeline.py` 的 capture→mid_term→episodic）或 event_log 写一条；**不要**绕过 `core/sandbox.get_paths()`，不要碰 `_sanitize_assistant_message` 之外的写法。

## 实现（决策定后）
1. 活动 `close` 端点（各 `*/close`）在 `summary_threshold` 满足时，调一个新 `activity_summary.generate_and_reflow(session)`：
   - 读该 session 的 activity transcript（`core/activity/transcript.py`）；
   - LLM 生成 ≤1–2 句关系/情绪向摘要；
   - 写入主记忆（episodic 或 event_log，单条），来源标记 `activity:{type}`。
2. 更新对应活动的 `MemoryPolicy`：把 `main_memory` 由 `none/deferred` 改为真正回流，并在 registry 注明。
3. 守住隔离：**过程中**仍不写 short_term/hidden_state；只有**结束摘要**这一条进主记忆。
4. 改了 registry/memory policy 后跑 contract smoke test + `docs/memory.md` 相关测试，必要时 `tests/run_eval.py`。

## 验收
- 完整玩一局（达阈值）并关闭后，主记忆多出一条"一起做过 X"的摘要；叶瑄之后聊天能自然提及。
- 草草开关（未达阈值）→ 不留记忆。
- 过程中不写 short_term/hidden_state（隔离仍在）。
- 摘要是情绪/关系向，不是棋谱/页码流水账。
- 相关记忆测试 + contract smoke test 通过。

## 备注
这项与 `cc-tasks/08-潜意识审计与流动概括条.md` 提到的 `08b-hidden-state-接现实写入` 是**两件独立的"记忆没接满"**：08b 是现实对话→隐性状态；本份是活动→主记忆。可分别决定做不做。
