- `ThemePicker.tsx` **已按 `manifest.base`（'light'/'dark'）分组**展示——天然适配"日间=light/夜间=dark"配对。
- `listThemes()` 合并内置（paper/dark/sakura-pink…）+ 磁盘 mod，含 CSS 安全检测（`cssGuard`）。
- `uiPreferences.ts`：`getUIPref/setUIPref`，键前缀 `emerald.ui.`，localStorage + JSON。
- **散落问题**：`ThemePicker` 现挂在两处——`ChatWindow.tsx:158`（设置里）和 `ActivitySettingsPage.tsx:127`。字体包选择器也在 `ChatWindow.tsx:179`。

## 目标（按用户拍板）
1. 把主题/字体/背景等"外观"项**聚合进一个独立栏目**（"扩展 / 外观"），不再散落。
2. **日夜双主题**：分别选一个日间主题、一个夜间主题，支持**热切换**（手动一键 + 可选按时间自动）。

## 实现

### 1. 日夜双槽 + 模式 `src/shared/theme/registry.ts`
新增偏好键（沿用 `chat.` 命名空间）：
- `chat.theme.day`（默认 `'paper'`）、`chat.theme.night`（默认 `'dark'`）、`chat.theme.mode`（`'manual' | 'auto'`，默认 `'manual'`）、`chat.theme.active`（`'day' | 'night'`，手动模式下记当前在哪个槽，默认 `'day'`）。
- 保留旧 `chat.theme` 作为"最终生效 id"的兼容镜像（`setTheme` 已在写它），避免破坏 `initTheme`/storage 同步。

新增函数：
```ts
export function getDayNight() { /* 读三个键 */ }
export function setSlotTheme(slot:'day'|'night', id:string)  // 存对应槽 + 若当前激活该槽则立即 setTheme(id)
export function setThemeMode(mode:'manual'|'auto')           // 存 mode；auto 时立即按时间应用
export function applyByMode()  // mode==auto → 按当前小时选 day/night 槽；manual → 用 chat.theme.active 槽；调 setTheme(选中id)
export function toggleDayNight() // 手动：active 在 day/night 间翻转 + setTheme(对应槽)；写回 chat.theme.active
```
- 自动时间规则：夜间 19:00–次日 6:00（小时阈值设常量，可调）。auto 模式下加一个轻量定时器（如每 10 分钟 `applyByMode()`，或监听跨过阈值）——简单起见用 `setInterval(applyByMode, 10*60*1000)`，在 `initTheme` 里按 mode 启停。
- `initTheme()` 改为：读 mode → `applyByMode()`（替代直接读单一 `chat.theme`）。保持 storage 跨窗口同步（active/slot 变化也广播：写这些键即可，已有 storage 监听；必要时扩展监听键名）。
- 所有失败 graceful fallback（找不到槽主题 → 回退 paper，registry 已有该兜底）。

### 2. 一键热切换入口 `Ribbon.tsx`
- 复用 `Ribbon.tsx:130` 的 `onThemeToggle`：接到 `toggleDayNight()`。点一下在日/夜两个槽之间即时切换（manual 模式）。图标可随 active 显示太阳/月亮。
- auto 模式下该按钮可临时覆盖（切到对方槽并暂置 manual，或下个 tick 再被 auto 纠正——二选一，注释说明，建议"点了就转 manual"以免和自动打架)。

### 3. 独立「外观 / 扩展」设置栏（聚合）
- 新建一个设置面板组件（如 `src/windows/chat/components/AppearancePanel.tsx`），集中放：
  - **日间主题**选择器 + **夜间主题**选择器（各用一个 `ThemePicker`，但分别绑 day/night 槽——给 `ThemePicker` 加 `slot?: 'day'|'night'` prop，`choose` 时调 `setSlotTheme(slot, id)` 而非全局 `setTheme`；不传 slot 时维持旧行为以兼容别处）。
  - **模式开关**：手动 / 自动（按时间）。
  - **字体包**（把 `ChatWindow.tsx:179` 那段迁来）。
  - 预留位：聊天背景图、主题自定义 CSS（v2 spec 的特性，本次可只占位或一并做）。
- `Ribbon` 的「偏好」(`onOpenPrefs`) 里新增「外观/扩展」入口，或在偏好面板里加一个分栏指向它。
- 从 `ChatWindow.tsx` / `ActivitySettingsPage.tsx` **移除散落的 ThemePicker**（或改为指向新面板），避免两份不一致。注意别破坏这两个文件其余设置项。

### 4. 兼容与回归
- 旧用户 localStorage 只有 `chat.theme` → `initTheme` 首次迁移：把它填进 `chat.theme.day`（若是 light）或 `chat.theme.night`（若是 dark），mode 默认 manual、active 对应。一次性、静默。
- 跨窗口：切日夜后所有已开窗口（chat/dream/activity/pet）应同步换肤——确认 storage 事件覆盖到新键（dream/activity 窗口若各自 `initTheme`，需都走 `applyByMode`）。

## 验收
- 外观栏里能分别设日间、夜间主题；Ribbon 一键在日/夜间热切换，立即生效、无闪白。
- 自动模式下，跨过夜间时间点会自动换到夜间主题（可临时改系统时间或把阈值调到当前分钟验证）。
- 多窗口同时开 → 切换后全部同步换肤。
- 旧 `chat.theme` 用户升级后不丢主题（迁移生效）。
- `npm run build` 通过，无 console 报错；找不到主题时回退 paper 不白屏。
