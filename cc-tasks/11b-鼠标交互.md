# CC-11b · 桌宠鼠标交互：害羞躲避 + 蹭 + 钉住（待办 g）

> 纯【前端 Emerald-client】（Tauri 窗口 API）。可与 07/09a/11a 并行。
> 移植 `_achive-Emeral-not/behavior/mouse_interact.py` + `sense/mouse_tracker.py` 的设计。

## 现状（已核对）
- `PetWindow.tsx` 左键 `getCurrentWindow().startDragging()` 可拖；**无任何鼠标感知/反应**。
- 老桌宠：`mouse_tracker.py`（每秒采全局鼠标坐标）+ `mouse_interact.py`（随机间隔触发"蹭"；shy 时窗口推离光标；按住 Ctrl `pinned` 不躲）。

## 实现 `src/windows/pet/`（新建 `usePetMouse.ts` hook）
1. **取鼠标坐标**：用 Tauri 的 cursor/window 位置 API（或轮询）。算光标与桌宠窗的距离。
2. **害羞躲避**：`shy` 状态下，光标进入桌宠窗一定半径 → 用 `getCurrentWindow().setPosition()` 把窗朝远离光标方向平移一小段（带缓动）。参考老桌宠的位移向量算法。
   - 边界保护：不要把窗推出屏幕（夹在可视区内）。
3. **蹭鼠标**：随机间隔（`bother_min/max`，可配）触发一次"靠近/蹭"的小动作（朝光标方向轻移 + 粒子反应），与 11a 的视觉联动。
4. **钉住**：按住 `hold_still` 键（默认 Ctrl）时 `pinned=true`，不躲也不蹭，只能被拖。
5. **与拖拽协调**：现有左键拖拽保留；建议 pinned 或特定区域才允许拖，避免"想拖却躲开"。`shy`/`pinned` 可由 mood 或后端 snapshot 驱动（如害羞情绪 → shy）。
6. 全局开关（设置项）允许关闭鼠标交互（有人会嫌烦）。

## 验收
- shy 时鼠标靠近桌宠会"躲"，不被推出屏幕。
- 随机会出现"蹭"动作；按住 Ctrl 时不躲、可稳定拖动。
- 关闭开关后桌宠不再躲/蹭。
- 无 console 报错；`npm run build` + Rust 编译通过。
