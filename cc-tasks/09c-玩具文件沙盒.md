## 已定决策
- 玩具文件放 **`data/very_formal_project/`**（名字是玩梗，实为玩具箱）。
- 工具 `dangerous=False`，靠 09b 的模式闸约束（safe 模式禁，danger 模式可）。

## 设计要点（安全第一）
- **绝不接受任意路径**，绝不把路径暴露给 LLM。LLM 只能用 `file_key`（枚举）。
- 后端常量映射 `file_key → 绝对路径`，全部落在 `get_paths()` 下的 `data/very_formal_project/`。
- 写入走 `core/safe_write.py`（原子写），文本-only，大小上限。

## 实现 `core/tool_dispatcher.py`（+ 可选 `core/tools/toybox.py`）

### 1. 沙盒路径 + 白名单
- `core/sandbox.py` / `core/data_paths.py` 加 `very_formal_project_dir()` 返回 `data/very_formal_project/`（经 `get_paths()`，禁止硬编码）。
- 常量白名单（举例，最终清单见下"待补"）：
```python
_TOYBOX_FILES = {
    "diary":     "我的非常正式的项目笔记.txt",
    "wishlist":  "愿望清单.md",
    "doodle":    "涂鸦板.txt",
}
```
  `file_key` 只能是这三个 key 之一；非法 key 直接拒绝。

### 2. 工具注册（category `desktop` 或新 `toybox`，受模式闸约束）
- `read_toy_file(file_key)`：读对应文件，不存在返回空内容提示。
- `write_toy_file(file_key, content, mode="overwrite"|"append")`：
  - 校验 `file_key ∈ _TOYBOX_FILES`；
  - 解析为 `very_formal_project_dir() / _TOYBOX_FILES[file_key]`，并**再次断言** resolve 后仍在该目录内（防 `..` / 软链穿越）；
  - 内容长度上限（如 4000 字）；
  - 走 `safe_write` 原子写；append 模式读后拼。
- 两工具都注册进 `_TOOL_REGISTRY`，带 `examples`/`keywords`（Hard Rule 2）；`category` 纳入 09b 的 `_MODE_RESTRICTED_CATEGORIES`（即 safe 模式禁、danger 模式可）。
- 给模型的描述要说清"这是你们的玩具项目文件，可以随便写写画画"，避免它当成系统文件。

### 3. 目录初始化
- 首次写入时若 `data/very_formal_project/` 不存在则创建（`mkdir parents=True`）。
