# CC-11a · 桌宠行为环（state 驱动）（待办 g）

> 纯【前端 Emerald-client】。可与 07/09a 并行。先读 `docs/pet-window-reference.md`。

## 现状（已核对）
- `PetWindow.tsx`：透明置顶窗 + `ParticleCanvas` + 气泡（`latestAssistantText` 更新弹 9s）+ 拖拽。
- 数据：`listenPetSnapshots` → `PetSnapshot{mood, presence, activityText, latestAssistantText, thinking, updatedAt}`。
- **缺**：气泡只在"有新回复"时被动弹，没有 idle/主动氛围；粒子/动作没充分被 mood/presence 驱动。

## 目标
没消息时也让人感到"它在"——**state 驱动，不是 message 驱动**（reference doc 核心原则）。前端只做视觉/氛围，**不硬编码台词**（说话仍走后端，见 11c）。

## 实现 `src/windows/pet/PetWindow.tsx` + `components/ParticleCanvas.tsx`
1. **粒子随情绪/在场**：把 `snapshot.mood / presence / thinking` 映射到 `ParticleCanvas` 的幅度/色相/速度（参考老桌宠 `particle.py` 的 `set_emotion/set_volume`）。
   - presence=away → 粒子收敛、变暗、降速；idle → 缓慢微动；active → 正常；thinking → 轻微聚拢/脉动。
2. **idle 氛围轮换**：无新消息时，按 mood 轮换顶部那行氛围词（现 `activity` 文案）或微动作，间隔随机化避免机械。**这些是氛围词，不是叶瑄发言**。
3. **mood 过渡平滑**：mood 变化时粒子参数做缓动过渡（避免突变），呼应老桌宠"呼吸感"。
4. 不引入新数据源；全部基于现有 snapshot。失败/字段缺失给安全默认（`DEFAULT_PET_SNAPSHOT` 已有）。

## 验收
- 长时间无消息时桌宠仍有可感的微动/呼吸，不死寂。
- 切到 away/idle/active/thinking，粒子与氛围有可感差异且过渡平滑。
- 无 console 报错；`npm run build` 通过。
