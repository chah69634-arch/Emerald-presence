# CC-08b · 隐性状态接现实写入（让潜意识真的随对话变化）（待办 b 的根因修复）

> 后端为主 + **设计决策先行**。先读 `docs/memory.md` + `docs/user_hidden_state_phase3.md` + `cc-tasks/08-潜意识审计与流动概括条.md`。
> ⚠ 这是**改记忆契约**的任务。当前隐性状态只被衰减/做梦驱动，本份把"现实对话"接成第三个写入源。先定"事件如何判定"，再接线。

## 现状（已核对，根因）
- `core/memory/user_hidden_state_integrator.py` 的 `integrate_event / integrate_impression / integrate_event_and_save / integrate_impression_and_save` **实现完整但全仓零调用**。
- 运行时唯一写 hidden_state 的：出梦 afterglow（`dream_exit_afterglow.integrate_afterglow_and_save`）、12h `apply_time_decay`、7d consolidate。
- → **不做梦就只朝基线衰减，所以面板看着静态**（待办 b 的根因）。

## 可用的写入原语（已核对）
- `RealityEventType`（`integrator.py:106`）：`SEEK_COMPANIONSHIP` / `RECEIVED_COMFORT` / `NO_INTERACTION`。
  - `integrate_event`：SEEK/RECEIVED → `touch_need.deficit` 放电（−）；NO_INTERACTION → 充电（+）。来源标 `UpdateSource.REALITY_BEHAVIOR`。
- `integrate_impression`：按 impression 调中期层（sensitivity.current 等），不碰长期基线。
- `integrate_event_and_save(uid, event_type, write_envelope, now, *, char_id)`：load → integrate → **仅当 `write_envelope.can_write_memory` 且 result.accepted 才原子落盘**。
- 长期字段（sensitivity.baseline / touch_need.baseline / embodied_ease / body_memory）由 `_assert_not_long_term` 守卫，**现实事件不可写**——别绕过。

## 设计决策（先定，本份的核心）
**问题：一个现实回合如何映射到 event_type / impression？** 这是关键决策，给三档方案，建议从轻到重：

- **方案 A（推荐起步，零额外 LLM）**：复用现有信号判定，不新增模型调用。
  - `NO_INTERACTION`：已有 scheduler 的"距上次互动时长"（见 presence_nag 用的同源信号）——长时间无互动时，由衰减/调度侧补一次 deficit 充电（其实 decay 已近似覆盖，可不重复）。
  - `RECEIVED_COMFORT` / `SEEK_COMPANIONSHIP`：用 pre-pipeline 已有的**话题标签**（`core/tag_rules.py` 的 tags）粗判——亲密/安抚类话题 → RECEIVED_COMFORT；用户主动靠近/求陪伴类 → SEEK_COMPANIONSHIP。无命中则不写。
- **方案 B**：在已有的 mid_term/impression 生成（post_process 慢队列里本就调 LLM 做压缩）里**顺带产出一个 impression 结构**喂 `integrate_impression_and_save`，不额外加调用。
- **方案 C（最重）**：专门加一个轻量分类探针。**不建议**，性价比低。

> 建议：**A + B 组合**——A 处理 touch_need（陪伴/冷落），B 借 mid_term 的 LLM 产出 impression 调 sensitivity。先做 A，跑通可见变化，再加 B。

## 实现（决策定后，照 trait_tracker 的接线范式）
> 参考 `CLAUDE.md` 记的 R8-B：`post_process()` 在每个 `can_write_memory=True` 的有效回合后入队 `trait_tracker_update` 慢任务。**照这个范式接 hidden_state。**

1. **判定**：在 `core/pipeline.py post_process()`（或其慢队列 handler）里，对有效 assistant 回合，按方案 A 用 tags/互动时长判出 `event_type`（可能为 None=不写）。
2. **入队慢任务**：`slow_queue.enqueue("hidden_state_integrate", {uid, char_id, event_type, ...})`，handler 调 `integrate_event_and_save(uid, event_type, write_envelope, now, char_id=char_id)`。
   - `write_envelope` 用该回合的（`can_write_memory` 必须为真才落盘——本就由 envelope 保证）。
   - 方案 B 的 impression：在 mid_term 慢任务产出 impression 后调 `integrate_impression_and_save`。
3. **并发**：写盘在 `uid_lock` / store 原子写内（`user_hidden_state_store.save_hidden_state` 已原子）；与 decay/consolidate 的 12h/7d 任务不冲突（不同触发，串行落盘）。
4. **source 守卫**：只走 `REALITY_BEHAVIOR` 中期层；**不得**触碰长期基线/body_memory（`_assert_not_long_term` 已挡，别绕）。
5. **可观测**：配合 `08` 的面板 delta 展示，接通后应能看到 `last_update_source` 出现 `reality_behavior`、deficit 随陪伴/冷落升降。

## 验收
- 一段亲密/被安抚的对话后，`touch_need.deficit` 下降且 `last_update_source=reality_behavior`；长时间冷落后回升。
- （做了 B）相关话题后 `sensitivity.current` 有合理微调，基线不动。
- 长期字段（baseline/body_memory）始终不被现实事件改写（`_assert_not_long_term` 守卫生效）。
- 仅 `can_write_memory=True` 的有效回合才写；测试/无效回合不写。
- 面板能看到潜意识"动起来"（不再只有 time_decay）。
- 后端 `pytest`（integrator 已有 phase3 测试；补一个"现实回合→deficit 变化"的接线用例）+ `tests/run_eval.py`（若动了 tag 判定）。

## 备注
与 `12c-活动记忆回流` 是**两件独立的"记忆没接满"**：本份是现实对话→隐性状态（中期层）；12c 是活动→主记忆（episodic）。互不依赖，可分别排期。
