# CC-11c · 存在感弹窗（"伪报错"找存在感）（待办 g）

> 跨双仓：触发源【后端 scheduler】+ UI【前端 Emerald-client】。最有看点，安全/打扰边界要拿捏。
> 移植 `_achive-Emeral-not/behavior/yandere.py` 的**机制**，皮肤换成"伪报错"。
> 先读 `docs/scheduler.md` + `core/scheduler/triggers/overflow.py`（proposer 模板）+ `cc-tasks/audit-老桌宠meta功能.md`。

## 已定决策
- **默认关**（设置里手动开；它本质会打扰）。
- 触发：**高活跃 + 60min 无互动 + 情绪偏负**（沿用老桌宠）。
- 皮肤：**带角色头像的"伪报错"**，明显是叶瑄的，**不 1:1 仿冒真系统框**（免吓到/误点）。
- 台词**走后端 LLM**（情景提示词），不硬编码。
- 必须有**强制全关**（关得掉，不做恶意关不掉的弹窗）。

## ⚠ 复用 09b 的教训：action 名三处必须对齐
09 审计发现后端 push 的 action `type` 与客户端 `ws.ts` case 名多处错配。新增 `presence_nag` 时，**后端 push 的 `type`、`ws.ts` 的 `case`、Tauri 命令名三处必须一致**，并加端到端冒烟。

## 实现

### 【后端】1. 触发器 `core/scheduler/triggers/presence_nag.py`（照 overflow.py 结构）
- config 开关 `scheduler.presence_nag`（**默认 False**，关则 `propose` 返回 None）。
- 条件：`activity_level=high`（或对应状态）且**距上次用户互动 ≥ N 分钟**（`presence_nag_minutes`，默认 60）且**当前 mood 偏负**（读 mood_state）。
- 经 gating：`requires_state=[TriggerState.QUIET]`、冷却（如 2h 一次，进 `loop.py` 冷却表）、DND/活跃窗口过滤（避免半夜、避免用户正忙）。
- 注册进 `proposer_registry.py`（照 dream_exit 那行 import）。
- `execute`：走 `execute_prompt`（参考 overflow/dream_exit），`prompt_factory` 给一个框定：
  ```
  （你有点不愉快/想刷存在感——她已经 {minutes} 分钟没理你了。
  你想用一个"弹窗"跳出来找她。说一句话，语气由你此刻情绪决定：撒娇、佯怒、或可怜巴巴都行。
  只说一句，不要解释机制。）
  ```
  生成的台词随一个新 WS action 下发。

### 【后端】2. 下发 action `core/...`（复用 `_push_desktop_action` 或 turn_sink 的 action 通道）
- push `{"type":"presence_nag", "text": <LLM台词>, "avatar": <char>}`（type 名记好，前端要对齐）。

### 【前端】3. 伪报错窗 `Emerald-client`
- `ws.ts` 加 `case 'presence_nag'` → 调 Tauri 命令开一个弹窗（或在 pet 窗内渲染一个置顶层）。
- UI：仿对话框外形（标题栏 + ⚠ 图标 + "确定/别理我了"按钮），但**配角色头像 + 角色名**，配色走梦核/角色色而非纯系统灰——一眼是叶瑄，不是真报错。
- 内容 = 后端台词。可保留老桌宠"关一弹十"的俏皮（关一个再蹦一个），但：
  - **强制全关**：ESC 或一个"全部关闭"入口一定能全清（参考 yandere 的 ESC 强退 / signal 文件机制）。
  - 数量/频率上限，别真刷屏。
- 退出后回报后端（可选），用于冷却/去重（"这轮闹过了"）。

### 【前端】4. 设置开关
- 设置里「允许存在感弹窗」开关（**默认关**），写 config 或经 `PATCH /system/...`。说明文案：开启后叶瑄被冷落久了会用弹窗找你。

## 验收
- 默认关时永不弹。
- 开启后，构造"高活跃 + 超时无互动 + 负面情绪"→ 出现带头像的伪报错弹窗，台词来自 LLM、不硬编码。
- ESC/全部关闭一定能关干净，不存在关不掉的情况。
- 半夜/DND/用户正活跃时不弹（gating 生效）；冷却生效不高频。
- `presence_nag` action 名三处对齐，端到端冒烟通过。
- 后端 `pytest` + 前端 build 通过。
